```rust
//! Strategy types: market state snapshots, entry decisions, and strategy identifiers.

use crate::{Asset, ContractPrice, Price, Side, Timeframe, WindowId};

#[cfg(feature = "std")]
use serde::{Deserialize, Serialize};

/// Identifies which strategy produced a trade. Used for per-strategy P&L attribution.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
#[cfg_attr(feature = "std", derive(Serialize, Deserialize))]
pub enum StrategyId {
    /// Buy both YES and NO when combined < $1.00.
    CompleteSetArb,
    /// Enter early (first 1-3 min) based on spot movement before market reprices.
    EarlyDirectional,
    /// Enter mid-window (min 5-10) on sustained momentum.
    MomentumConfirmation,
    /// Buy opposite side to lock guaranteed profit on losing position.
    HedgeLock,
}

impl core::fmt::Display for StrategyId {
    fn fmt(&self, f: &mut core::fmt::Formatter<'_>) -> core::fmt::Result {
        match self {
            Self::CompleteSetArb => write!(f, "Arb"),
            Self::EarlyDirectional => write!(f, "EarlyDir"),
            Self::MomentumConfirmation => write!(f, "Momentum"),
            Self::HedgeLock => write!(f, "Hedge"),
        }
    }
}

/// Complete market state snapshot passed to strategies on every tick.
///
/// Contains both spot exchange data and Polymarket orderbook data.
/// Contract prices are `Option` because Polymarket data may be unavailable
/// (e.g., during backtest with sparse trade data).
#[derive(Debug, Clone, Copy)]
pub struct MarketState {
    /// Which asset.
    pub asset: Asset,
    /// Window duration type.
    pub timeframe: Timeframe,
    /// Window identifier.
    pub window_id: WindowId,
    /// Spot price at window open.
    pub window_open_price: Price,
    /// Latest spot price from exchange.
    pub current_spot: Price,
    /// `abs(current - open) / open` as fraction.
    pub spot_magnitude: f64,
    /// Direction of spot movement.
    pub spot_direction: Side,
    /// Seconds elapsed since window opened.
    pub time_elapsed_secs: u64,
    /// Seconds remaining until window closes.
    pub time_remaining_secs: u64,
    /// Polymarket best ask for Up contract (what you'd pay to buy Up).
    pub contract_ask_up: Option<ContractPrice>,
    /// Polymarket best ask for Down contract (what you'd pay to buy Down).
    pub contract_ask_down: Option<ContractPrice>,
    /// Polymarket best bid for Up contract (what you'd get selling Up).
    pub contract_bid_up: Option<ContractPrice>,
    /// Polymarket best bid for Down contract (what you'd get selling Down).
    pub contract_bid_down: Option<ContractPrice>,
}

impl MarketState {
    /// Best ask for the side matching spot direction.
    #[must_use]
    pub fn direction_ask(&self) -> Option<ContractPrice> {
        match self.spot_direction {
            Side::Up => self.contract_ask_up,
            Side::Down => self.contract_ask_down,
        }
    }

    /// Best ask for the side opposite to spot direction.
    #[must_use]
    pub fn opposite_ask(&self) -> Option<ContractPrice> {
        match self.spot_direction {
            Side::Up => self.contract_ask_down,
            Side::Down => self.contract_ask_up,
        }
    }
}

/// A decision to enter a position, produced by a [`Strategy`].
#[derive(Debug, Clone, Copy)]
pub struct EntryDecision {
    /// Which side to buy.
    pub side: Side,
    /// Maximum price we're willing to pay.
    pub limit_price: ContractPrice,
    /// Confidence level (0.0-1.0). Influences Kelly sizing.
    pub confidence: f64,
    /// Which strategy produced this decision.
    pub strategy_id: StrategyId,
}
```

```rust
//! Strategy trait: the core abstraction for all trading strategies.

use pm_types::{EntryDecision, MarketState, StrategyId};

/// A trading strategy evaluates market state and optionally produces an entry decision.
///
/// Strategies are stateless — all state is in [`MarketState`].
/// The strategy engine runs all strategies on every tick and picks the best decision.
pub trait Strategy {
    /// Unique identifier for this strategy.
    fn id(&self) -> StrategyId;

    /// Evaluate whether current market state warrants an entry.
    /// Returns `None` if no trade should be taken.
    fn evaluate(&self, state: &MarketState) -> Option<EntryDecision>;
}
```

- [ ] **Step 2: Write CompleteSetArb (arb.rs)**

```rust
//! Strategy A: Complete-Set Arbitrage.
//! Buy both YES and NO when combined cost < $1.00.

use pm_types::{ContractPrice, EntryDecision, MarketState, Side, StrategyId};
use crate::strategy::Strategy;

/// Configuration for complete-set arbitrage.
pub struct CompleteSetArb {
    /// Maximum combined cost of YES + NO to trigger (e.g., 0.98).
    pub max_combined_cost: f64,
    /// Minimum profit per share to bother (e.g., 0.015).
    pub min_profit_per_share: f64,
}

impl Strategy for CompleteSetArb {
    fn id(&self) -> StrategyId { StrategyId::CompleteSetArb }

    fn evaluate(&self, state: &MarketState) -> Option<EntryDecision> {
        let ask_up = state.contract_ask_up?.as_f64();
        let ask_down = state.contract_ask_down?.as_f64();
        let combined = ask_up + ask_down;

        if combined >= self.max_combined_cost {
            return None;
        }

        let profit = 1.0 - combined;
        if profit < self.min_profit_per_share {
            return None;
        }

        // Buy the cheaper side as primary entry.
        // Executor must handle the arb case: buy BOTH sides.
        let (side, price) = if ask_up <= ask_down {
            (Side::Up, ask_up)
        } else {
            (Side::Down, ask_down)
        };

        Some(EntryDecision {
            side,
            limit_price: ContractPrice::new(price)?,
            confidence: 1.0,
            strategy_id: StrategyId::CompleteSetArb,
        })
    }
}
```

Tests: combined < threshold → fires, combined > threshold → None, missing prices → None.

- [ ] **Step 3: Write EarlyDirectional (early.rs)**

```rust
//! Strategy B: Early Directional.
//! Enter in the first 1-3 minutes when spot has moved but contract hasn't repriced.

use pm_types::{EntryDecision, MarketState, StrategyId};
use crate::strategy::Strategy;

/// Configuration for early directional entry.
pub struct EarlyDirectional {
    /// Maximum seconds after window open to consider entry (e.g., 180).
    pub max_entry_time_secs: u64,
    /// Minimum spot magnitude to trigger (e.g., 0.001 = 0.1%).
    pub min_spot_magnitude: f64,
    /// Maximum contract price to pay (e.g., 0.60).
    pub max_entry_price: f64,
}

impl Strategy for EarlyDirectional {
    fn id(&self) -> StrategyId { StrategyId::EarlyDirectional }

    fn evaluate(&self, state: &MarketState) -> Option<EntryDecision> {
        // Only in early window.
        if state.time_elapsed_secs > self.max_entry_time_secs {
            return None;
        }

        // Need minimum spot movement.
        if state.spot_magnitude < self.min_spot_magnitude {
            return None;
        }

        // Get the ask price for the direction we want to buy.
        let ask = state.direction_ask()?.as_f64();

        // Don't overpay.
        if ask > self.max_entry_price {
            return None;
        }

        Some(EntryDecision {
            side: state.spot_direction,
            limit_price: state.direction_ask()?,
            confidence: 0.7,
            strategy_id: StrategyId::EarlyDirectional,
        })
    }
}
```

Tests: early + magnitude + cheap price → fires, too late → None, price too high → None, magnitude too small → None.

## Task 6: pm-executor — Realistic Backtest Engine (rewrite)

**Files:**
- Rewrite: `polymarket/crates/pm-executor/src/backtest.rs`

- [ ] **Step 1: Rewrite backtest with real contract prices**

The new backtest:
1. Iterates ticks chronologically
2. For each tick, constructs `MarketState` with:
   - Spot data from Binance (already have)
   - Contract price from Polymarket trade data OR contract price model
3. Runs `StrategyEngine::evaluate_all()` against the state
4. Records entries per strategy at the ACTUAL contract price
5. At window close, resolves all positions
6. Produces per-strategy P&L breakdown

Key changes from v1:
- `MarketState` replaces the old `(tick, window, market_price)` tuple
- Contract prices come from real data, not simulation
- Multiple strategies can fire in the same window (different strategies, different entries)
- `TradeRecord` now carries `strategy_id` for attribution

New `BacktestResult`:
```rust
pub struct BacktestResult {
    pub trades: Vec<TradeRecord>,
    pub summary: TradeSummary,
    pub per_strategy: HashMap<StrategyId, TradeSummary>,
    pub final_balance: f64,
}
```
